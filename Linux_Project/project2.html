L2 — Ubuntu Server Hosting & Troubleshooting

Real-world Ubuntu server hosting tasks including NGINX, Node.js apps, port binding, SSL, firewall rules, DNS, and deep troubleshooting of deployment failures.

Project Overview

This project covers the deployment, configuration, and debugging of Ubuntu servers hosting real applications. It includes NGINX reverse proxy setup, Node.js hosting, service log debugging, and solving port conflicts, firewall issues, file permission errors, and process-level problems.

Key Server Hosting Tasks

  • Installing & configuring NGINX as a reverse proxy
  • Deploying and managing Node.js web applications
  • Fixing port binding errors (EADDRINUSE)
  • Setting up systemd services for auto-starting apps
  • Configuring ufw firewall rules for ports 80/443/3000
  • Troubleshooting HTTP/HTTPS routing issues
  • Debugging server logs using journalctl & syslog
  • Fixing 502/504 gateway errors from NGINX
  • Restarting crashed services and clearing stale locks

Major Troubleshooting Scenarios

  • NGINX failing due to misconfigured server blocks
  • Node.js service not starting due to missing environment variables
  • Fixing “address already in use” port conflicts
  • Resolving permission issues for service scripts
  • Correcting DNS resolution errors for hosted sites
  • Fixing blank-page errors due to routing issues
  • Rebuilding systemd service files with correct ExecStart paths

Achievements

  • Successfully deployed a stable Ubuntu production server
  • Configured NGINX reverse proxy with custom routing
  • Documented key troubleshooting workflows
  • Mastered systemd, logs, firewalls, and service management
