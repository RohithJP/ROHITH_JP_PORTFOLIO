Architecture Workflow

Detailed Explanation

This project demonstrates how production servers securely forward logs to a central Rsyslog instance for monitoring, troubleshooting, and long-term retention.

  • Client Systems: Applications and EC2 instances generate logs (auth, nginx, syslog, app logs).
  • Rsyslog Forwarder: Each node is configured with *.* @@<server-ip>:514 using TCP for reliability.
  • Central Server: Collects all remote logs, applies filters, templates, and categorises logs using dynamic file structure (/var/log/remote/$HOSTNAME/).
  • Storage: Logs stored locally and optionally pushed to an S3 bucket for long-term archiving.
  • Monitoring: Alerts created for SSH failures, Nginx errors, and application-specific patterns.
