Project 3 — Centralized Log Management (Rsyslog + CloudWatch)

Implemented secure, centralized log collection across multiple EC2 instances using Rsyslog, directory templating, and AWS CloudWatch integration.

View Repository →

Overview

The objective was to centralize application and system logs from multiple EC2 servers (Web & DB) into a single Log Collector instance using Rsyslog. Logs were further streamed to AWS CloudWatch for real-time monitoring, alerting, and auditing.

Workflow Diagram

AWS Web EC2 → AWS DB EC2 → → Rsyslog Log Collector (EC2) → → CloudWatch Log Group → → Metrics & Alarms

Tech Stack

  • EC2 (Ubuntu 22.04)
  • Rsyslog TCP/UDP
  • CloudWatch Agent
  • Security Groups / VPC Networking
  • Linux log forwarding

Setup & Implementation

1. Log Collector EC2 Setup

  • Created directory /var/log/remote with syslog ownership
  • Enabled TCP & UDP listeners (port 514)
  • Used hostname-based directory templates for log separation
  • Restarted and enabled Rsyslog service

2. Web & DB Server Setup

  • Configured Rsyslog to forward all logs using TCP
  • Used private IPs for intra-VPC security
  • Restarted Rsyslog on all nodes

3. Testing & Verification

  • Tested log forwarding using logger -p local0.info
  • Verified folder auto-creation per EC2 hostname
  • Validated real-time logs in CloudWatch log group

Troubleshooting

  • Fixed Rsyslog path permissions preventing log creation
  • Resolved UDP packet drops via Security Group adjustments
  • Corrected Rsyslog template errors for directory creation
  • Used journalctl -u rsyslog -f for real-time debugging
  • Ensured CloudWatch agent had correct log file paths

What I Achieved

  • Centralized Linux logs across multiple EC2 servers
  • Real-time CloudWatch monitoring & metric-based alarms
  • Structured log storage based on hostname
  • Improved auditability and troubleshooting visibility
  • Hardened log pipeline security using VPC internal routing
