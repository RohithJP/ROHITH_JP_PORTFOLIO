Secure File Transfer & Storage Workflow
SFTP → EC2 SFTP server → Automated upload → AWS S3 (versioned). Secure, auditable, and automated file sharing across teams.

Architecture

[India Dev Team] → SFTP → EC2 SFTP Server → Automated Upload → S3 Bucket (versioning) → UK Sales Team

• EC2 acts as the SFTP endpoint with chrooted users.
• Files placed into user uploads are synced to an S3 bucket via cron/script using an attached IAM role.
• S3 versioning is enabled for recovery and auditability.

SFTP EC2 S3 IAM Versioning

Project Overview

Goal: Build a secure file transfer system using OpenSSH SFTP on EC2 with chrooted users and automated upload to an S3 bucket with versioning enabled. The system ensures secure daily product-file exchange between India development and UK sales teams with recoverability and audit trails.

Setup & Configuration

Step 1
Launch EC2
Ubuntu instance with attached IAM role for S3 access.
Step 2
Create SFTP Users
`adduser sftp_user` + create `/home/sftp_user/uploads` and set correct ownership/permissions.
Step 3
Chroot Configuration
Edit `/etc/ssh/sshd_config` — `Subsystem sftp internal-sftp` + `Match User sftp_user` with `ChrootDirectory /home/sftp_user` & `ForceCommand internal-sftp`.
Step 4
AWS CLI & Upload Script
Install AWS CLI v2; create `/home/sftp_user/upload_s3.sh`: 
#!/bin/bash
aws s3 sync /home/sftp_user/uploads s3://projrct4-buck --exact-timestamps
Add to crontab: `0 * * * * /home/sftp_user/upload_s3.sh`
Step 5
S3 Versioning
Enable versioning in the bucket: `aws s3api put-bucket-versioning --bucket projrct4-buck --versioning-configuration Status=Enabled`

Troubleshooting & Lessons Learned

  • SSH restart errors — usually misconfigured `sshd_config` or wrong directory ownership (chroot dirs must be root-owned).
  • AWS CLI access denied — attach IAM role to EC2 instead of using static keys.
  • Cron job not running — ensure script has shebang, full paths, executable bit, and correct owner.
  • Version restore testing — use S3 console or `aws s3api list-object-versions` then copy older version as needed.

Techniques Learned

  • Implementing chrooted SFTP users to isolate file uploads.
  • Using IAM roles to securely grant EC2 S3 access (no credentials on disk).
  • Automating periodic syncs via cron and ensuring atomic file operations.
  • Enabling S3 versioning for safe rollbacks and retention policies.
