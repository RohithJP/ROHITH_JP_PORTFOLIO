Automated Static Website Hosting
EC2 + Nginx + S3 sync automation using userdata & Bash for continuous deployment.

Architecture

EC2 (Nginx) → Automated Sync → S3 Storage

• EC2 hosts the website
• S3 stores the versioned static files
• Bash script syncs updates automatically

EC2 Nginx S3 IAM Role Bash

Project Overview

Designed an automated static hosting setup where EC2 runs Nginx as the serving layer and S3 stores the source files. A Bash script + cron job syncs updated HTML/CSS files to S3 every hour. The instance uses an IAM role for secure S3 access.

Setup & Steps

Step 1
Launch EC2
Ubuntu + IAM Role + Nginx install.
Step 2
S3 Bucket
Private bucket for master files.
Step 3
Bash Script
aws s3 sync → /var/www/html/
Step 4
Cron Job
Runs script hourly to auto-deploy updates.

Troubleshooting

  • S3 access denied → Fixed by attaching correct IAM role to EC2.
  • Nginx not serving → Permissions corrected for /var/www/html/.
  • Cron not running → Corrected script ownership & added full paths.

Techniques Learned

  • Userdata automation for instant provisioning.
  • S3 sync patterns for efficient deployments.
  • IAM role vs access keys for secure automation.
