PHP Website Deployment with MySQL RDS
End-to-end deployment of a dynamic PHP web application on AWS EC2 using NGINX, PHP-FPM, and Amazon RDS MySQL. Includes debugging 502/500 errors, DB connectivity, and Linux permissions.

Architecture

GitHub → EC2 (NGINX + PHP-FPM) → RDS MySQL → Browser / Users

• EC2 hosts NGINX + PHP-FPM
• RDS hosts MySQL with secure inbound rules
• Website cloned directly from GitHub
• Permissions fixed to avoid 403/500 errors
• PHP app connected to RDS endpoint

NGINX PHP-FPM RDS EC2 GitHub

Project Overview

The objective was to deploy a dynamic PHP website ("sms\_kaliswari\_college") using AWS EC2 with NGINX + PHP-FPM and connect it to a MySQL RDS instance. The project focused on secure hosting, database connectivity, and resolving 502/500 application errors.

Setup & Configuration

Step 1
Launch EC2
Ubuntu 22.04 / t2.micro. Install NGINX & basic packages.
Step 2
Install NGINX
sudo apt install nginx -y Remove default site and configure custom virtual host.
Step 3
Install PHP & PHP-FPM
sudo apt install php php-fpm php-mysql -y
Step 4
Clone Website From GitHub
git clone <repo> /var/www/html/mywebsite
Step 5
Create MySQL RDS Database
Create DB + user + import SQL dump.
Step 6
Connect PHP to RDS
Update db.php with RDS endpoint, username, password.

Troubleshooting & Fixes

  • 502 Bad Gateway — PHP-FPM socket mismatch. Fixed in NGINX config.
  • 500 Internal Server Error — Missing PHP modules / DB errors.
  • RDS Access Denied — EC2 not allowed in RDS SG. Added SG rule.
  • NGINX default page showing — default site not removed.
  • SSH permission denied — incorrect PEM permissions.

Techniques Learned

  • Creating and configuring NGINX virtual hosts.
  • Fixing PHP-FPM socket & permission issues.
  • Importing MySQL dumps to RDS from EC2.
  • Debugging 502/500 using NGINX + PHP logs.
  • Linux file permissions (www-data ownership).
