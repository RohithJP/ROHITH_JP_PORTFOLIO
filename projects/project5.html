Hosting a PHP Website on AWS with MySQL RDS

Deploying a dynamic PHP website from GitHub → EC2 → RDS → Public website

Project Overview

Goal: Host a production-ready PHP website on AWS EC2, connect it to RDS MySQL, and make it publicly accessible.

Scenario:

  • Clone website from GitHub
  • Deploy on EC2 using NGINX + PHP-FPM
  • Create RDS MySQL and connect it
  • Import SQL database

This project demonstrates real-world Linux, DB, web server, and cloud deployment skills.

Learning Objectives

  • Launch & configure EC2
  • Install & configure NGINX
  • Install PHP & PHP-FPM
  • Create RDS MySQL DB
  • Import SQL data
  • Connect PHP → RDS
  • Troubleshoot 502 / 500 errors
  • Manage Linux permissions, security groups, networking

Architecture

Flow:

[GitHub Repository] → Clone → [EC2 Ubuntu] → [NGINX + PHP-FPM] → [RDS MySQL] → [Users]

  • EC2 (Ubuntu 22.04)
  • NGINX Web Server
  • PHP-FPM
  • AWS RDS (MySQL)
  • Linux permissions & AWS security groups

1⃣ EC2 Instance Setup

chmod 400 mykey.pem
ssh -i "mykey.pem" ubuntu@<EC2-PUBLIC-IP>

sudo apt update && sudo apt upgrade -y
sudo apt install git unzip curl -y

2⃣ NGINX Installation & Configuration

sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx

Remove default site:

sudo rm /etc/nginx/sites-enabled/default

Create custom site:

sudo nano /etc/nginx/sites-available/mywebsite

NGINX config:

server {
    listen 80;
    server_name _;

    root /var/www/html/mywebsite;
    index index.php index.html index.htm;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;
    }

    location ~ /\.ht {
        deny all;
    }
}

Enable site:

sudo ln -s /etc/nginx/sites-available/mywebsite /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

3⃣ PHP & PHP-FPM Installation

sudo apt install php php-fpm php-mysql php-mbstring php-xml -y
php -v
sudo systemctl enable php8.2-fpm
