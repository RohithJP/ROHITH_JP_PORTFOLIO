Multi-Tier WordPress Deployment
A complete 3-tier WordPress infrastructure built using NGINX reverse proxy, Apache/PHP web tier, and MySQL database tier — deployed with strong networking, tier-based security, and backup automation.

Workflow

Project Overview

This project implements a production-style multi-tier WordPress application inside AWS, separating the reverse proxy, web server, and database server into isolated subnets. The deployment follows real-world practices such as bastion access, tier-based security groups, automated backups, and hardened SSH authentication.

Goal

Deploy a secure 3-tier WordPress infrastructure using AWS EC2, following real-world reliability, scalability, and security standards.

Architecture

Internet → Nginx Reverse Proxy → Apache/PHP Web Server → MySQL Database

  • Public Subnet (10.0.1.0/24) — Reverse Proxy
  • Private Subnet (10.0.2.0/24) — Web Tier
  • Private Subnet (10.0.3.0/24) — Database Tier

VPC & AWS Console Setup

Step-by-step AWS manual configuration:

  • Create VPC: 10.0.0.0/16
  • Create subnets: public, web-private, db-private
  • Attach Internet Gateway
  • Create route tables for public & private subnets
  • Launch NAT Gateway for outbound updates
  • Create tier-based security groups

EC2 Instance Deployment

  • reverse-proxy — Ubuntu 24.04 (public subnet)
  • web-server — Ubuntu 24.04 (private subnet)
  • database-server — Ubuntu 24.04 (private subnet)

Troubleshooting & Fixes
