01 — Multi-Tier WordPress Deployment

A complete 3-tier architecture using Nginx reverse proxy, Apache/PHP web tier, and MySQL database tier. Deployed with strict networking, SG rules, backups, and security hardening.

Workflow

Project Overview

This project delivers a production-style multi-tier WordPress application across isolated subnets, a hardened network perimeter, and a fully documented operational workflow.

Goal

Deploy a secure 3-tier WordPress infrastructure using AWS EC2, following real-world reliability, scalability, and security standards.

Architecture

Internet → Reverse Proxy (Nginx) → Apache/PHP Web Server → MySQL Database

  • Public Subnet (10.0.1.0/24) — Reverse Proxy
  • Private Subnet (10.0.2.0/24) — Web Tier
  • Private Subnet (10.0.3.0/24) — Database Tier

VPC & AWS Console Setup

Step 1: Create VPC

  • Name: wordpress-vpc
  • CIDR: 10.0.0.0/16

Step 2: Create Subnets

  • Public Subnet: 10.0.1.0/24
  • Web Private Subnet: 10.0.2.0/24
  • DB Private Subnet: 10.0.3.0/24

Step 3: Internet Gateway

  • Create IGW → Attach to VPC

Step 4: Route Tables

  • Public: 0.0.0.0/0 → IGW
  • Private: 0.0.0.0/0 → NAT Gateway

Step 5: NAT Gateway

Required for package installations in private subnets.

Step 6: Security Groups

  • proxy-sg: HTTP/SSH from public
  • web-sg: HTTP/SSH from proxy
  • db-sg: MySQL from web tier

EC2 Instance Deployment

Reverse Proxy

  • Ubuntu 24.04 — Public Subnet
  • Nginx reverse proxy

Web Server

  • Ubuntu 24.04 — Private Subnet
  • Apache2 + PHP
  • WordPress installation

Database Server

  • Ubuntu 24.04 — Private Subnet
  • MySQL secured + DB user for WordPress

Troubleshooting

  • SSH failures → fixed ProxyJump configuration
  • MySQL remote access → updated bind-address + grants
  • WordPress 403 errors → corrected Apache permissions
  • Nginx 502 errors → upstream proxy correction

Techniques Learned

  • VPC networking & subnet zoning
  • Nginx reverse proxy load direction
  • Apache virtual hosts & PHP config
  • MySQL remote authentication security
  • SSH bastion-style access (ProxyJump)

Achievements

  • Automated MySQL backups implemented
  • Complete troubleshooting documentation
